Q. What is the difference between TCP and UDP? Give examples of where each is used.
    TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are both transport layer protocols, but they differ in behavior:

    TCP is connection-oriented and provides reliable communication. It ensures that data is delivered in sequence and without loss or duplication. It's slightly slower due to this overhead.

    Example: Email, web browsing (HTTP/HTTPS), and file transfers (FTP) use TCP.

    UDP is connectionless and does not guarantee delivery or order, but it's faster and has low latency.

    Example: Video streaming, online gaming, and VoIP (Voice over IP) use UDP, where speed is more important than reliability.


Q. Why is TCP considered reliable? Explain the specific mechanisms TCP uses to ensure reliability.
    TCP is considered reliable because it uses several mechanisms to ensure that data is delivered accurately and in order:

    Sequencing:
    Each data packet is assigned a sequence number. This helps the receiver reorder packets if they arrive out of order.

    Acknowledgment (ACKs):
    After receiving a packet, the receiver sends back an acknowledgment number to confirm successful delivery. If this ACK is not received by the sender, it assumes the packet was lost.

    Retransmission:
    If the sender does not receive an acknowledgment within a certain timeout period, it retransmits the packet.

    Flow Control (using Windowing):
    TCP uses a sliding window protocol to ensure that the sender does not overwhelm the receiver with too much data at once.

    Error Checking:
    TCP also includes checksums to detect corrupted data. If a checksum fails, the packet is discarded and retransmitted.

Q. What is the difference between a public IP address and a private IP address? Can you give some examples of private IP ranges?
    Public IP address is an IP address that is globally unique and can be accessed over the internet. It is assigned by your ISP and is used to identify your device on the internet.

    Private IP address is used within local networks (LAN), like in homes, schools, or offices. These are not routable on the public internet and are reserved for internal use only.
        Most home Wi-Fi networks use IPs from the 192.168.x.x range.

Q. What is NAT and how does it allow multiple devices to use the internet?
    NAT (Network Address Translation) is a process used by routers to allow multiple devices on a private network to access the internet using a single public IP address.

    At home, your phone, laptop, and smart TV may all have private IPs like 192.168.1.x. When any of them accesses the internet, the router uses NAT to translate their private IP to the one public IP assigned by your ISP.

    NAT helps solve the problem of IPv4 address shortage, because not every device needs a public IP. It also adds a layer of security, since internal devices are not directly exposed to the public internet.

Q. What is a deadlock? How can it be avoided?
    A deadlock is a situation in an operating system where two or more processes are waiting for each other to release resources, and none of them can proceed.
    üìå Example:
        Process A holds Resource R1 and waits for R2.
        Process B holds Resource R2 and waits for R1.
        Neither can proceed ‚Äî deadlock occurs.
    
    üîÑ Four Conditions for Deadlock (Coffman Conditions):
        Mutual Exclusion: Resources are non-shareable (only one process can use a resource at a time).
        Hold and Wait: A process holds at least one resource and waits for more.
        No Preemption: A resource can't be forcibly taken from a process.
        Circular Wait: A set of processes are waiting for each other in a circular chain.
            If all four are true at the same time ‚Äî deadlock can occur.
    
    üõ°Ô∏è How to Avoid Deadlock?
        1. Deadlock Prevention (Break at least one Coffman condition):
            Mutual Exclusion: Not always possible (e.g., a printer).
            Hold and Wait: Request all resources at once (but this can reduce efficiency).
            No Preemption: Allow preemption (forcefully take a resource if needed).
            Circular Wait: Impose a fixed ordering on resource requests.

        2. Deadlock Avoidance (Use algorithms to make safe decisions):
            Banker‚Äôs Algorithm: Works like a bank lending system.
            Only allocates resources if it leads to a safe state.
            Needs advance knowledge of maximum resource needs.

Q. What is the difference between Paging and Segmentation?
    ‚úÖ Paging:
        Paging is a memory management scheme that divides logical memory into fixed-size pages and physical memory into fixed-size frames.
        It helps eliminate external fragmentation but may cause internal fragmentation.
        It is based on the physical structure of memory.
        Each process uses a page table to map logical pages to physical frames.
        Logical address = <Page Number, Offset>.

    ‚úÖ Segmentation:
        Segmentation divides memory into variable-size segments based on logical divisions like code, data, stack, etc.

        It may cause external fragmentation, but no internal fragmentation.

        It reflects the logical structure of the program.

        Each process has a segment table to map segment numbers to physical addresses.

        Logical address = <Segment Number, Offset>.
