3. What is a Replica Set in MongoDB?
    A replica set is a group of MongoDB servers that work together to ensuring data redundancy and high availability 
    It consists of one primary node and one more secondary node that replicate the primary node's data

    Read Operation:
        -> By default, all read operations are directed to the primary node to ensure data consistency.
        -> Secondary nodes can also handle read operations if explicitly configured using a read preference
            eg.
            // Default read (from primary)
                db.collection.find({ age: { $gte: 30 } });

            // Read from secondary
                db.collection.find({ age: { $gte: 30 } }).readPref("secondary");
    
    Write Operation:
        -> All write operations are handled by the primary node
        -> The primary node logs the operation in its oplog (operation log), and the secondaries replicate the change asynchronously.

4. How does MongoDB handle data consistence?
    
    1.Write consistency
        Levels: 
        w: 1 (default): Acknowledgment from the primary node only.
        w: "majority": Acknowledgment from a majority of nodes in the replica set.
        w: 0: No acknowledgment required (fire-and-forget mode).

            eg. db.collection.insertOne({ name: "Alice", age: 25 }, { writeConcern: { w: "majority" } });

    2. Read consistency
        Levels:
        local (default): Returns data from the nodeâ€™s memory, not guaranteeing replication.
        majority: Ensures the data read is acknowledged by a majority of the nodes.
        linearizable: Guarantees the read operation reflects the most recent write.
        available: Returns data even if the node is not in sync.

        eg. db.collection.find({ name: "Alice" }).readConcern("majority");
    
    3.  Replica Sets
        The primary node handles all writes.
        The secondary nodes replicate data from the primary asynchronously.
        If the primary fails, a new primary is elected to ensure continuity.

    4. Transaction Support
        MongoDB supports ACID transactions for multi-document operations.
        Transactions ensure atomicity, consistency, isolation, and durability across multiple documents or collections.
