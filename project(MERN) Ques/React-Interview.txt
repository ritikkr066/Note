4. Explain the component lifecycle in React ?

    The component lifecycle  is sequence of events that happen to a component from its creation(mounting) to its removal(unmounting).

    1. mounting : constructor()  render()  componentDidMount()
    2. updating : shouldComponentUpdate() render() componentDidUpdate()
    3. unmounting : componentWillUnmount()

    Mounting : when the component is created and inserted into the DOM.
    Updating : when the component's state or props change
    Unmounting : Shutting down the process

5. What is context API , and how is it different from props ?
    The Context API  allows us to share state and data across componenet without actually  passing  props down manually at every level.
    
    it helps to manage global state and avoids " prop drilling" where we have to pass data through many layers of components.

6. How can you optimise Performance within a React application?

    Optimizing Performance in a React Application:
    1.Use React.memo:- Prevents unnecessary re-renders by memoizing functional components.
        eg. const OptimizedComponent = memo(({ data }) => <div>{data}</div>);
    
    2.Use useCallback and useMemo:-
        useCallback: Memoizes functions to avoid re-creation during re-renders.
        useMemo: Memoizes computed values.
        eg. const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
            const memoizedCallback = useCallback(() => doSomething(a), [a]);

    3.Code-Splitting and Lazy Loading:-Load components and modules only when needed.

    4.Avoid Anonymous Functions in JSX:-Avoid inline functions that can cause unnecessary re-renders.
        // Bad
        <button onClick={() => handleClick()}>Click</button>
        // Good
        const handleClick = useCallback(() => {...}, []);
        <button onClick={handleClick}>Click</button>

9. What is server-side rendering , how is it beneficial?
    Server-side rendering(SSR) is a Technique where a web pages is rendered on the server before it sent to the user's browser.

    Instead a sending a blank HTML with javascript code that builds the page on client's side,the server generates the HTML with content already includes and sends it to the browser.
        eg. Times of India uses SSR . all content already loaded
            Instagram uses CSR(client-side rendering based on user preference)

    Advantages:-
        -> Faster Initial load : since the HTML is pre-rendered on the server, the browser can display the content quickly, improving the preceived load times for users.

        -> Better SEO : search engines can easily index the content of server-side rendered pages because they don't need to execute javascript to see the content.

        -> Improved Performance :- Since most of the rendering is done on the server, client devices don't need to do as much work to render the page which is helpful for users on slower or less powerful devices.

        -> with SSR, users can interact with the content more quickly because they don't have to wait for the javascript to fully load before seeing anything

10. What do you understand by Redux in React ?
    In React, components manage their own internal state, but as an application grows, managing shared state between multiple componenets can become complex. This is where Redux helps by providing a single global state that any componenet can access
