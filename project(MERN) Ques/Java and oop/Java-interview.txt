1. what is the difference between JDK,JRE,JVM in java?

    In Java, the JDK, JRE, and JVM are key components of the Java development and execution environment, and each has a specific role:

    JVM (Java Virtual Machine) is the engine that actually runs Java bytecode. It provides platform independence and handles tasks like memory management, garbage collection, and security. It's responsible for converting bytecode into machine-specific code.

    JRE (Java Runtime Environment) is a package that provides the libraries, JVM, and other components to run Java applications. It does not include tools for development like compilers or debuggers.

    JDK (Java Development Kit) is the full development kit. It includes everything in the JRE, plus development tools like the Java compiler (javac), debugger, and other tools required to write, compile, and debug Java programs.

        #"JVM runs the code, JRE provides the environment to run it, and JDK provides the tools to develop it."

7. What is an abstract class in java?
    An abstract class in Java is a class that cannot be instantiated â€” it's meant to be extended by other classes.
        It can contain:
            Abstract methods (methods without a body)
            And also concrete methods (methods with a body)

            Eg.
            abstract class Animal {
                abstract void makeSound(); // abstract method

                void sleep() { // concrete method
                    System.out.println("Sleeping...");
                }
            }

        any subclass must implement the abstract method
            class Dog extends Animal {
                void makeSound() {
                    System.out.println("Dog barks");
                }
            }
    An abstract class is used when we want to define common behavior and force subclasses to implement specific methods.
8. What is an Interface?

    An interface in Java is a completely abstract type that is used to define a contract â€” it only contains method signatures and constants, but no method bodies
        All methods in an interface are:
            public
            abstract (by default)
    
    In Java, a class cannot inherit from more than one class because Java does not support multiple inheritance with classes â€” this is to avoid ambiguity issues like the Diamond Problem.
    
    However, Java does support multiple inheritance with interfaces. That means a class can implement multiple interfaces at the same time.

            interface Printable {
                void print();
            }
            interface Scannable {
                void scan();
            }
            class PrinterScanner implements Printable, Scannable {
                public void print() {
                    System.out.println("Printing...");
                }
                public void scan() {
                    System.out.println("Scanning...");
                }
            }

     if Java did support multiple class inheritance, the syntax might look like C++
            class A {
                void show() {
                    System.out.println("From class A");
                }
            }
            class B {
                void show() {
                    System.out.println("From class B");
                }
            }
            // Hypothetical syntax (this is NOT allowed in Java)
            class C extends A, B {
                // Ambiguity: which show() should be used?
            }

    Compiler get confused, which show mehod should be called.
    So, Java uses interfaces to achieve multiple inheritance safely, because interfaces do not carry implementation (unless explicitly marked as default), so there's no ambiguity.

10. what is the use of 'this' keyword in java ?
    In Java, the this keyword is a reference variable that refers to the current object of the class.
        Itâ€™s mainly used for three purposes:

        ðŸ”¹ 1. To refer to instance variables when local variables have the same name:
            class Student {
                String name;
                Student(String name) {
                    this.name = name;  // 'this.name' refers to instance variable, 'name' is constructor parameter
                }
            }

         2. To call current class methods or constructors:
            this.display();       // Calls a method of the current class  
            this(10);             // Calls another constructor in the same class (constructor chaining)

        3.  To pass the current object as a parameter:
            class A {
                void show(A obj) {
                    System.out.println("Object received");
                }

                void callShow() {
                    show(this);  // Passing current object to method
                }
            }

24. Explain the importance of synchorised keyword?
    synchronized is used to make sure that when multiple threads try to use the same resource, they donâ€™t mess each other up. It locks the code so only one thread can run it at a time, which keeps things safe and accurate.

    with synchorise(Thread-safe)

    class Counter {
        int count = 0;

        synchronized void increment() {
            count++;
        }
    }

    public class SynchronizedExample {
        public static void main(String[] args) throws InterruptedException {
            Counter counter = new Counter();

            Thread t1 = new Thread(() -> {
                for (int i = 0; i < 1000; i++) {
                    counter.increment();
                }
            });

            Thread t2 = new Thread(() -> {
                for (int i = 0; i < 1000; i++) {
                    counter.increment();
                }
            });

            t1.start();
            t2.start();
            t1.join();
            t2.join();

            System.out.println("Final Count (with synchronized): " + counter.count);
        }
    }

    ##Final Count (with synchronized): 2000

28. Explain the concept of autoboxing and unboxing?
    Autoboxing
        The automatic conversion of a primitive type into its corresponding wrapper class.
    
    Unboxing 
        Converts an object of a wrapper class back into its corresponding primitive type.

    eg. List<Integer> numbers = new ArrayList<>();
        numbers.add(5);  // Autoboxing: 5 (int) -> Integer
        int result = numbers.get(0);  // Unboxing: Integer -> int

29. What is the purpose of trasient keyword in serialisation ?
   The transient keyword tells Java not to save that variable when we serialize an object. Itâ€™s useful for hiding passwords or temporary data during serialization.

   "Both static and transient variables are not serialized, but for different reasons:
   ðŸ”¹ static keyword:
        static means the variable belongs to the class, not to any specific object.
