-----------------  Common Networking Commands: ----------------

1. ping
    Purpose: Checks connectivity between your system and a remote host.
    How it works: Sends ICMP echo requests and waits for a reply.
    Usage: ping google.com

2. netstat
    Purpose: Displays network connections, routing tables, and open ports.
    Helps you see which applications are using which ports.
    Usage:  netstat -an

3. tracert (Windows) / traceroute (Linux)
    Purpose: Shows the route (hops) a packet takes to reach a destination.
    Useful to find delays or broken links in a network path.
    Usage: tracert google.com

4. ipconfig (Windows)
    Purpose: Displays IP address, subnet mask, default gateway, and more.
    Also used to renew or release DHCP IP addresses.
    Usage: ipconfig

5. nslookup
    Purpose: Queries DNS records to resolve domain names to IP addresses.
    Helps verify DNS configuration and issues.
    Usage: nslookup google.com

6.  route
    Purpose: Displays or modifies the IP routing table.
    Shows how data is routed to different networks.
    Usage:  route print       # View routing table
            route add ...     # Add a static route

7. pathping (Windows only)
    Purpose: Combines ping + tracert â€” shows route + packet loss and latency per hop.
    Takes more time but gives detailed diagnostics.
    Usage: pathping google.com

8. hostname
